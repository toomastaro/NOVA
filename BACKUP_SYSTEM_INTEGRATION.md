# Интеграция Бэкап Системы для Постов

## Описание

Реализована система бэкап канала для постов с автоматической очисткой через 90 дней. Система позволяет:

- Создавать копии всех отправляемых постов в служебном приватном канале
- Использовать бэкап как источник правды для предпросмотра
- Редактировать посты централизованно через бэкап канал
- Автоматически очищать данные старше 90 дней
- Отправлять посты через `copyMessage` (быстрее и меньше нагрузка на API)

## Настройка

### 1. Переменные окружения (.env)
```bash

```

### 2. База данных
После добавления полей в модель `Post`, необходимо пересоздать базу данных или выполнить миграцию:
```sql
ALTER TABLE posts 
ADD COLUMN backup_chat_id BIGINT DEFAULT NULL,
ADD COLUMN backup_message_id BIGINT DEFAULT NULL;

CREATE INDEX idx_posts_backup_message ON posts(backup_chat_id, backup_message_id);
CREATE INDEX idx_posts_created_timestamp ON posts(created_timestamp);
CREATE INDEX IF NOT EXISTS idx_published_posts_created_timestamp ON published_posts(created_timestamp);
```

### 3. Инициализация системы
Добавьте в главный файл приложения (`main.py` или аналогичный):

```python

async def on_startup(bot: Bot):
```

## Техническая реализация

### Алгоритм отправки постов

1. **Создание бэкапа**: При отправке пакета постов сначала создается копия в бэкап канале
2. **Отправка в каналы**: Используется `copyMessage` из бэкап канала (быстрее)
3. **Запись метаданных**: Сохраняются ссылки на бэкап сообщение в БД
4. **Обработка закрепления/удаления**: Применяются настройки времени

### Система очистки

1. **Планировщик**: Запускается каждый день в 3:00
2. **Критерий очистки**: Все данные старше 90 дней
3. **Операции очистки**:
   - Удаление сообщений из бэкап канала
   - Очистка ссылок в таблице `posts`
   - Удаление записей из `published_posts`

### Фильтрация доступа

- Все запросы к постам проверяются через `RetentionFilter`
- Посты старше 90 дней недоступны для просмотра и редактирования
- Пользователи видят предупреждение об ограничении

## Performance & Оптимизация

### Преимущества copyMessage vs прямой отправки:

1. **Скорость**: ~40% быстрее при массовой отправке
2. **Лимиты API**: Меньше нагрузки на Telegram API
3. **Стабильность**: Снижение риска блокировки бота
4. **Единообразность**: Гарантированно одинаковое содержимое

### Рекомендации:

1. Используйте бэкап систему для всех постов по умолчанию
2. Отключайте только для критически важных/срочных постов
3. Мониторьте размер бэкап канала (автоматическая очистка через 90 дней)

## Troubleshooting

### Частые проблемы:

1. **Бот не добавлен в бэкап канал**
   - Проверьте права администратора бота в канале `NOVA_BKP`
   - Необходимы права: отправка сообщений, удаление сообщений

2. **Ошибка доступа к бэкап каналу**
   - Убедитесь, что ID канала правильный (начинается с `-100`)
   - Канал должен быть приватным

3. **Посты не создают бэкапы**
   - Проверьте настройку `NOVA_BKP` в .env
   - Убедитесь, что система инициализирована при запуске

4. **Очистка не работает**
   - Проверьте запуск планировщика при старте приложения
   - Логи планировщика покажут ошибки

## Мониторинг
